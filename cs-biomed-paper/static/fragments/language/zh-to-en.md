# Language: Chinese-to-English (default)

Use when the user's notes are Chinese, mixed Chinese-English, or organized as lab notes.

## Translate intent, not syntax

Chinese academic notes pack background, motivation, method, and implication into one long sentence.
Before drafting English, decompose each note into: **claim / evidence / condition / comparison /
implication / limitation**. Then write English in the order the *section* requires, not the order of
the Chinese sentence.

## Common repairs

| Chinese-draft pattern | English repair |
|---|---|
| Broad importance before a clear object | Name the system/clinical problem earlier |
| Method list before the research gap | Move the gap before the method |
| `显著提高 / 明显改善` without a baseline | Add the comparator or soften the verb |
| `首次 / 创新性` without scope | Replace with a bounded novelty claim |
| Mechanism inferred from correlation | Use `suggests` / `is consistent with`, or ask for evidence |
| Results mixed with implications | Observation in Results, meaning in Discussion |
| Repeated topic noun where English uses a pronoun | Replace or elide |
| Strings of short clauses joined by commas | Split or add explicit connectives |
| `大量实验表明` | State the actual experiments and numbers |

## Terminology discipline

Lock one canonical English term per concept (method/model/dataset/metric) and reuse it everywhere;
don't let translation introduce synonyms. Keep an internal Terminology Ledger.

## Output convention

Give the **polished English first**, then a brief **Chinese note** explaining the major structural,
hedging, and terminology choices — so the author can verify intent without re-reading the English.
Never change a scientific claim while translating.
