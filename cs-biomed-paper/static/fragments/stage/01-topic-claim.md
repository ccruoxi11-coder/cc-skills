# Stage 01 — Topic & novelty positioning

**Goal:** turn an idea into a defensible claim with a clinical/biological *and* a methodological
reason to exist.

## Steps

1. Write the one-sentence argument (claim–evidence–boundary, see `core/stance.md`).
2. Name the **gap** twice: the *clinical/biological* gap (what decision or understanding is currently
   limited) and the *methodological* gap (why existing methods don't close it). A CS-only gap is a
   common desk-reject reason at biomedical venues.
3. List 2–4 **contributions** as verifiable statements, each tied to evidence you actually have.
   Order: clinical/biological impact first for medical venues; methodological first for ML venues.
4. State the **boundary** explicitly: population, modality, sites, what it does *not* claim.
5. Sanity-check **stakes**: who acts differently if this is true? If you cannot name the decision or
   the biological insight, the topic is not yet positioned.

## Common pitfalls

- "Novel architecture" with no clinical question → reframe around the decision it improves.
- Claiming generalizability from a single-center dataset → cap the claim or add external data (→ 04).
- "First to apply X to Y" as the whole contribution → weak; add *why it matters* and *what it reveals*.
- Hidden scope creep: the abstract promises more populations than the data cover.

## Output

The one-sentence argument, the dual gap, the contribution list, the boundary, and any
`[待补充：临床终点 / 目标人群 / 外部数据]`.
