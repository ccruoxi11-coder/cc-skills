# Section: Abstract

## Default structure (structured or unstructured)

`background/clinical need → gap → what we did (approach + data) → key quantified result (with the main
metric + uncertainty) → external validation → conclusion + boundary`. If the venue uses a structured
abstract (Background/Methods/Results/Conclusions), map to its headings.

## Rules

- Lead with the **clinical/biological need** for medical venues; with the methodological gap for ML
  venues.
- Put the **headline number with its uncertainty** (e.g. AUC 0.xx, 95% CI x–x) and name the cohorts
  (internal + external n). Every number here must match the body exactly.
- State the boundary in the last sentence. No claim the body doesn't support.
- Draft the abstract **last**, after the argument is fixed.

## Pitfalls

- Numbers in the abstract that differ from the tables (the most common self-audit failure).
- Method-heavy abstract with no result, or result with no comparator/uncertainty.
- Over-claimed generalization from a single cohort.

Mark `[待补充：主指标及 95% CI / 外部队列 n]` for anything not yet available.
